# System Architecture: Meaningful URL & State Hydration Implementation

This document details the architectural blueprint and implementation steps for migrating the Wedding Payment platform from URL query parameters to a secure, Edge-routed State Hydration pattern on AWS.

## 1. Data Contract (The Source of Truth)

We define a strict JSON schema for the payload stored in DynamoDB. The `slug` acts as the Partition Key.

**DynamoDB Table:** `Gift4Event-Configurations`
*   **Partition Key:** `slug` (String)

**JSON Payload Example:**
```json
{
  "slug": "ron-and-maya-lvyz6b",
  "coupleNames": "רון ומאיה",
  "wazeLink": "https://waze.com/ul/...",
  "bankDetails": {
    "beneficiaryName": "רון כהן",
    "bankName": "בנק הפועלים",
    "branchNumber": "123",
    "accountNumber": "987654321"
  },
  "bitLinks": [
    "https://bitpay.co.il/app/...",
    "https://bitpay.co.il/app/..."
  ],
  "payboxLinks": [
    "https://payboxapp.page.link/...",
    "https://payboxapp.page.link/..."
  ],
  "createdAt": "2026-05-17T10:28:00Z"
}
```

## 2. Edge Routing (CloudFront Function)

A lightweight JavaScript function deployed to the CloudFront Distribution. It intercepts the request before it reaches the S3 origin.

**Routing Logic:**
1. If the request is for the root `/` or `index.html`, serve it normally.
2. If the request has a file extension (e.g., `.js`, `.css`, `.png`), serve it normally.
3. If the request is a clean path (e.g., `/ron-and-maya-lvyz6b`), internally rewrite the request to `/payment.html`.

**CloudFront Function Code:**
```javascript
function handler(event) {
    var request = event.request;
    var uri = request.uri;

    // Ignore root, index, API, or static assets
    if (uri === '/' || uri === '/index.html' || uri.includes('.') || uri.startsWith('/api/')) {
        return request;
    }

    // Rewrite clean slugs to the payment shell
    request.uri = '/payment.html';
    return request;
}
```

## 3. Backend Creation API (AWS Lambda + API Gateway)

An API Gateway endpoint (e.g., routed via CloudFront `/api/create`) handles the form submission from the link generator.

*   **Action:** `index.html` sends a POST request containing the form data.
*   **Processing:** A Lambda function validates the data, generates the unique slug, and writes the JSON payload to DynamoDB via a `PutItem` operation.
*   **Response:** Returns the generated slug URL to the creator.

### Slug Uniqueness Strategy: Base Slug + Creation Timestamp

To guarantee that the meaningful URL (slug) is globally unique and avoids collisions, the Lambda function employs a deterministic timestamp suffix.

*   **Approach:** Transliterate the couple's names into an English base slug. Append a compact string derived from the exact date and time the link is generated by converting the current Unix millisecond timestamp into a short Base36 string.
*   **Example:** `ron-and-maya-lvyz6b` (where `lvyz6b` represents the exact creation millisecond). 
*   **Architectural Advantage:** This guarantees 100% collision-free URLs by design, as no two links for the same couple will be generated at the exact same millisecond. It requires zero database lookups (no retry loops) and eliminates the need for purely random string generation logic.

## 4. Client-Side Hydration (`payment.html`)

The `payment.html` shell relies on client-side JavaScript to fetch its state dynamically based on the URL.

**Execution Flow:**
1.  **Path Extraction:** On DOM load, extract the slug from `window.location.pathname` (e.g., `"ron-and-maya-lvyz6b"`).
2.  **API Fetch & Edge Caching:** Make an asynchronous HTTP GET request to the read API: `/api/config/{slug}`.
    *   **The Cache Layer:** This request is routed through CloudFront. CloudFront is configured to aggressively cache responses for the `/api/config/*` path.
    *   **Cache Hit:** If another guest has recently visited this link, CloudFront returns the JSON instantly from the Edge (sub-millisecond latency, zero DynamoDB reads).
    *   **Cache Miss:** If not cached, the request falls through to the Lambda function, which performs a fast `GetItem` from DynamoDB, returns the JSON, and CloudFront caches it for all future guests.
3.  **UI Hydration:** Inject the received JSON payload into the DOM elements (updating `document.title`, `couple-names`, bank details, and payment links dynamically).
4.  **Error Handling:** If the API returns a 404 (invalid or deleted slug), the JavaScript catches the error and renders a user-friendly "Event Not Found" state within the shell.

## 5. Required Infrastructure Updates (Terraform)

To support this architecture, the following resources must be added to the Terraform state:
*   `aws_dynamodb_table`: Partition key `slug` (String), provisioned capacity or on-demand.
*   `aws_api_gateway_rest_api`: For routing `/api/create` and `/api/config/{slug}`.
*   `aws_lambda_function`: The Node.js/Python logic for handling reads and writes.
*   `aws_iam_role`: Granting Lambda permissions to read/write to the specific DynamoDB table.
*   `aws_cloudfront_function`: The edge routing script.
*   `aws_cloudfront_distribution` modifications: Attach the CloudFront function to viewer requests and map `/api/*` to the API Gateway origin.
